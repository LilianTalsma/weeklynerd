Leerdoelen

Leerdoelen Februari

  • Mijn creatieve ideeën kunnen realiseren op het web.

    Aan het begin van deze minor wist ik nog vrijwel niks van code schrijven, met alleen de basis had ik veel moeite met het realiseren van mijn ideeën. Door alle projecten leerde ik steeds meer over niet alleen de code zelf maar ook het stylen daarvan.

  • Mezelf uitdagen en niet opgeven als het moeilijk word.

    Als ik 1 ding heb gedaan de afgelopen weken is het wel niet opgeven. Soms had ik veel tegenslagen en snapte ik er even helemaal niks van. Gelukkig stond er een team aan docenten en andere leerlingen klaar om mij te helpen waar dat nodig was. Ik liep vaak aan tegen problemen met bijvoorbeeld CSS waar ik helemaal niet uitkwam maar op het moment dat ik mijn laptop door het raam wou gooien verander je 1 ding waardoor alles weer werkt. Ik ben erachter gekomen dat code schrijven een grote puzzel is en dat zou waarschijnlijk ook nog wel lang zo blijven, alleen leer ik hem wel elke keer snellet op te lossen.

  • Aan het einde van de minor trots zijn op wat ik heb geleerd en bereikt.

    We zijn nog niet aan het einde van de minor maar ik ben nu al mega trots op wat ik heb bereikt in deze weken. Ik had nooit verwacht dat ik zo 'gemakkelijk' een website in elkaar kon zetten. Ookal gaat het nog niet helemaal perfect lukt het mij om steeds sneller mijn ideeen te realisereren.

Leerdoelen Meesterproef

  • Meer leren over CSS en JavaScript.

    Ik wil beter worden in het gebruiken van geavanceerde CSS-technieken, zoals animaties en grid-layouts. Ook wil ik meer leren over JavaScript, zodat ik mijn websites interactiever kan maken en me niet meer 'beperk met alleen css.

  • Ik heb tijdens de meesterproef zeker meer geleerd over css vooral op gebied van styling voor een ander soort websit dan ik normaal doe. Ik heb ook geleerd hoe ik een horizontale scroll kan faken waardoor het lijkt of de pagina horizontaal scrollt terwijl het eigenlijk verticaal is. Ook heb ik veel geleerd over het werken met javascript, vooral met het maken van een API en het werken met de data die daaruit komt. Ik heb nu een veel beter beeld van hoe je een API kan gebruiken en hoe je de data die daaruit komt kan gebruiken in je website en op een dynamische manier kan laten zien. Ook heb ik een klasje gevolgd in GSAP en hoe ik deze toepas in een website. Ik vond dit erg intressant en ga dit zeker vaker gebruiken in mijn projecten. Ik vind dat ik dit leerdoel heb behaald omdat ik elke week ben blijven leren over nieuwe dingen die ik nog niet wist.

  • Beter worden in werken met Github.

    Ik wil beter worden in het niet weggooien van mijn code met github. In het aankomende project gaan we waarschijnlijk ook in github werken. In het verleden ging dit niet altijd goed. Het is vaker gebeurd dat ik bijvoorbeeld met de hackaton code van mijn team genoten overschreef of mijn hele project kwijt raaktte. Dit is dus een goed punt om op te letten en beter mee om te leren gaan.

  • Ik snap een stuk meer van Github, hoe ik het werk georganiseerd moet houden en hoe ik in mijn eigen branche moet werken en niet in de main. Dit zodat er geen problemen ontstaan met het overschrijven van elkaars code. Maar mijn grootste accomplishment is dat ik niks heb verwijderd!

  • Websites maken die toegankelijk zijn voor iedereen.

    Ik wil ervoor zorgen dat mijn websites bruikbaar zijn voor iedereen, ongeacht hun apparaat of beperking. Door de lecture van Nienke ben ik erg geintresseerd geraakt in dat vakgebied en ik denk dat daar nog veel te halen valt. Ik wil dus bij de meesterproef hier ook veel aandacht aan besteden naast de opdracht zelf.

  • Ik vind toegankelijkheid echt mega belangrijk. Met de meesterproef heb ik dan ook goed gekeken naar hoe iemand die bijvoorbeeld geen muis kan gebruiken door de pagina's manouvreerd. Zo hebben alle blokjes van de verhalen een tabindex gekregen zodat je er makkelijk doorheen kan navigeren met de tab knop. Ook heb ik ervoor gezorgd dat alle afbeeldingen een alt tekst hebben. Alle teksten zijn ook goed leesbaar en hebben een goede kleurcontrast. Zo hoop ik dat de website toegankelijk is voor iedereen, en blijf ik dit ook in de toekomst doen.

  • Plezier houden in code schrijven.

    Ik merk dat ik steeds meer plezier krijg in het schrijven van code. Ik vind het leuk om te zien dat ik steeds meer kan en dat ik steeds sneller mijn ideeën kan realiseren. Ik wil dit plezier ook in de meesterproef behouden en niet alleen maar bezig zijn met de opdracht zelf maar ook met het leren van nieuwe dingen.

  • Ik heb na de meesterproef en de minor nog steeds heel veel plezier in code schrijven!

Wat vond ik van de minor

De minor web heeft mij echt verbaasd. Ik begon met vrijwel geen kennis van code. En ik dacht eerlijk gezegd ook in de eerste weken van wat heb ik mijzelf aan gedaan. Maar narmate het half jaar verder ging begon ik steeds meer te begrijpen van de code en hoe het werkt, en het ook steeds leuker te vinden. Als ik kijk naar de eerste weken van de minor en nu, dan zie ik echt een groot verschil. Mijn WAFS site en deze van de meesterproef laten dan denk ik ook wel een groot verschil zien. Ik ben er ook tijdens deze minor achtergekomen dat ik accessibility heel intressant vind en wil hier nu graag later wat mee doen. Ik heb absoluut geen spijt van deze minor en denk dat ik absoluut een betere programmeur, maar ook designer ben geworden. Ik ben benieuwd of ik uiteindelijk nog wat met code ga doen maar denk dat het sowieso geen kwaat kan om dit te hebben moeten leren. Mijn grootste accomplishment is dat ik nu zelfstandig een website kan bouwen inplaats van alleen in figma te ontwerpen. Uiteindelijk ben ik heel trots op wat ik heb bereikt en geleerd in deze minor.
